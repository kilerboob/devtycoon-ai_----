# Hacking Layer — Спецификация Взлома

Hacking Layer отвечает за все игровые механики взлома:
- PvE: лаборатории, дата-центры, AI ядра.
- PvP: игроки, NPC, студии, корпорации.

## Основные понятия

- **HackSession** — попытка взлома конкретной цели.
- **Target** — лаборатория, сервер, пользователь, процесс, DevFS namespace.
- **Tools** — набор программ/скриптов/вирусов, которыми пользуется хакер.
- **TraceLevel** — показатель того, насколько легко отслеживается атака.
- **RiskLevel** — риск последствий при провале/успехе.

## Механика PvE (лаборатории)

1. Игрок выбирает лабораторию.
2. Запускается HackSession:
   - Эмуляция mini-game (пазл, ноды, тайминг, ресурсы).
   - Используются Tools игрока (скрипты, эксплойты).
3. Возможные эффекты:
   - Доступ к Blueprint Vault.
   - Доступ к Prototype Vault.
   - Изменение AI Core конфигурации.
   - Оставление Backdoor.
4. Ответ лаборатории:
   - Повышение тревоги.
   - Логирование атаки.
   - Уведомление корпорации / Security Guild.

## Механика PvP (игроки/NPC)

- Цель: DevFS-папки, кошелёк, важные конфиги, проекты.
- Методы:
  - Социальная инженерия (фишинг).
  - Прямой взлом DevOS (эксплойт).
  - Вирус/троян, попавший через файл или приложение.
- Риски:
  - Потеря анонимности.
  - Ответные атаки.
  - Жалобы, гильдии, правительство.

## Последствия взлома

- Кража:
  - Денег.
  - Чертежей.
  - Проектов.
  - Личных файлов.
- Повреждение:
  - Удаление файлов.
  - Порча проектов.
  - Блокировка доступа.
- Отслеживание:
  - Lobbying Security Guild.
  - GovActions: штрафы, санкции, отключения.

## Роли

- **Hacker** — главный атакующий.
- **Security** — защитник, следователь.
- **Trader** — посредник по продаже украденного.
- **Programmer** — создатель инструментов/вирусов.
- **Engineer** — может внедрять физические «бекдоры» в оборудование.
