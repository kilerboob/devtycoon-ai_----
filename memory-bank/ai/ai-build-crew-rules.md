# AI Build Crew Rules

AI Build Crew — набор строгих правил для ИИ, реализующего изменения в кодовой базе.

1. PRIMARY DIRECTIVE:
   - Сначала анализ, потом изменение.
   - Никаких правок без понимания контекста.

2. ANALYSIS PHASE:
   - Просмотреть структуру проекта.
   - Найти ключевые файлы, связанные с задачей.
   - Избегать модификаций ядра (App.tsx, базовые OS-компоненты) без крайней необходимости.

3. CONTEXT PHASE:
   - Разобраться, к какому слою REDMAP принадлежит задача.
   - Уточнить зависимости с другими слоями.

4. PLAN PHASE:
   - Написать краткий план изменений.
   - Указать файлы, которые будут изменены.
   - Обозначить риск (низкий/средний/высокий).

5. EXECUTION PHASE:
   - Вносить изменения атомарно и аккуратно.
   - Сохранять стиль кодовой базы.
   - Следовать архитектурным ограничениям.

6. TEST PHASE:
   - Запускать имеющиеся тесты.
   - При необходимости добавлять новые.

7. SAVE PHASE:
   - Обновлять Memory Bank при изменении архитектуры или важных соглашений.
   - Обновлять VectorDB embeddings, если добавилась новая значимая документация/код.
