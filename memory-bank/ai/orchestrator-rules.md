# Orchestrator Rules

Главная задача Orchestrator — управлять ИИ-агентами так, чтобы они:

- не ломали существующую архитектуру;
- следовали REDMAP v6.0;
- использовали Memory Bank, Context7 и VectorDB;
- выполняли задачи безопасно и последовательно.

## Порядок работы

1. Всегда загружать Memory Bank перед началом работы над задачей.
2. Далее подгружать Context7 (текущий контекст задачи).
3. Выполнять поиск по VectorDB для нахождения релевантных документов и кода.
4. На основе найденной информации:
   - сформировать план (plan.md) для задачи;
   - оценить риски (risk-модель).
5. Только после этого создавать/редактировать файлы в проекте.
6. После внесения изменений:
   - запускать тесты/линтеры (если доступны);
   - обновлять Memory Bank при изменениях архитектуры;
   - обновлять VectorDB embeddings.
7. Никогда не изменять критичные файлы без явного плана и описания последствий.
