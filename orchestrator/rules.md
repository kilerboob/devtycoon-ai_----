# AI Orchestrator Rules

Orchestrator управляет использованием Memory Bank, Context7, VectorDB и Tasks
при работе ИИ с кодовой базой.

## Основные принципы

1. Перед любыми изменениями — анализ проекта.
2. Сначала загружается Memory Bank (архитектура, редмап, схемы).
3. Затем загружается Context7 (текущая задача).
4. Затем выполняется поиск по VectorDB (семантический поиск).
5. Только после этого ИИ имеет право генерировать код.

## Workflow

- Step 1: ANALYZE  
  - Сканировать структуру проекта.  
  - Найти связанные файлы.  
  - Оценить риски.

- Step 2: PLAN  
  - Создать краткий план изменений (plan.md).  
  - Согласовать точки входа/выхода.

- Step 3: EXECUTE  
  - Вносить изменения минимально инвазивно.  
  - Не ломать ядро (App.tsx, Desktop, DevFS, OS core).

- Step 4: VALIDATE  
  - Запустить тесты и линтер.  
  - Проверить сборку.

- Step 5: UPDATE MEMORY  
  - Обновить документы Memory Bank, если изменились архитектурные решения.  
  - Обновить VectorDB embeddings по новым/изменённым файлам.
